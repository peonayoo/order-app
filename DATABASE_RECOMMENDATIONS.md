# 데이터베이스 대안 추천

PostgreSQL 대신 사용할 수 있는 데이터베이스 옵션들을 추천합니다.

## 🥇 1순위: SQLite (개발용)

### 장점
- ✅ **설치 불필요** - Mac에 기본 포함되어 있음
- ✅ **설정 없음** - 파일 하나만으로 작동
- ✅ **가볍고 빠름** - 개발/테스트용으로 완벽
- ✅ **백업/이동 쉬움** - 파일만 복사하면 됨
- ✅ **SQL 표준 준수** - PostgreSQL과 거의 동일한 SQL 문법

### 단점
- ❌ 동시 접속 제한 (읽기는 많지만 쓰기는 1개)
- ❌ 프로덕션 대용량에는 부적합

### 설치 방법
```bash
# Mac에는 이미 설치되어 있음
sqlite3 --version
```

### 사용 방법
```bash
# Node.js 패키지 설치
npm install better-sqlite3
# 또는
npm install sqlite3
```

## 🥈 2순위: MySQL / MariaDB

### 장점
- ✅ 널리 사용되는 안정적인 데이터베이스
- ✅ 설치 및 관리가 비교적 쉬움
- ✅ 좋은 성능과 확장성
- ✅ Mac에서 Homebrew로 쉽게 설치 가능

### 단점
- ❌ 별도 설치 필요
- ❌ 서버 관리 필요

### 설치 방법 (Homebrew)
```bash
# MySQL 설치
brew install mysql
brew services start mysql

# 또는 MariaDB
brew install mariadb
brew services start mariadb
```

### Node.js 패키지
```bash
npm install mysql2
```

## 🥉 3순위: Supabase (클라우드)

### 장점
- ✅ PostgreSQL 기반이지만 클라우드로 관리
- ✅ 무료 플랜 제공
- ✅ 설정이 거의 없음
- ✅ 자동 백업 및 스케일링
- ✅ 실시간 기능 포함

### 단점
- ❌ 인터넷 연결 필요
- ❌ 클라우드 서비스 의존

### 사용 방법
1. https://supabase.com 에서 계정 생성
2. 프로젝트 생성
3. 연결 정보 복사하여 `.env`에 설정
